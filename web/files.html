RWTester
software for testing a random walk

Analysis data files

This tutorial describes and explains what data files are produced as a result of the random walk analysis. In the following text template is used as a default graph name or an argument of -template option.

Graph analysis

Results of the graph analysis are these files:

  • template_info.txt
  • template_deg.txt
  • template_in.txt (directed graphs only)
  • template_out.txt (directed graphs only)

template_info.txt
This file contains summary information about the graph. Here (template_info.txt) is an example of such a file (this file was produced by RWTester version 0.1alpha).

template_deg.txt
This file contains data dependency how many vertices have the given degree. Here (template_deg.txt) is an example of such a file. If we use this file as an input for ChartGenerator this image (template_deg.png) will be the result. Meaning of the degree for directed graphs is sum of both in and out degrees.

template_in.txt
The same as above but for in-degree only. This file is produced for directed graphs only.

template_out.txt
The same as above but for out-degree only. This file is produced for directed graphs only.

Random walk analysis

Results of the random walk analysis are these files:

  • template_coverage.txt
  • template_degree_visited.txt
  • template_in_degree_visited.txt (directed graphs only)
  • template_out_degree_visited.txt (directed graphs only)
  • template_degree_time.txt
  • template_in_degree_time.txt (directed graphs only)
  • template_out_degree_time.txt (directed graphs only)
  • template_degree_time_length.txt
  • template_in_degree_time_length.txt (directed graphs only)
  • template_out_degree_time_length.txt (directed graphs only)

template_coverage.txt
This is data dependency in what time the given graph coverage was achieved. For example 50% of a graph coverage means that exactly one half of all vertices were at least once visited by the running random walk. Let's have a look at this line: 

50    129680

This means that 50% graph coverage was achieved after 129680 steps of the random walk.

Here (template_coverage.txt) is an example of such a file. Notice that the border of the random walk was 90% graph coverage. So the coverage higher than 90% has the time equal to zero. ChartGenerator result for this file is here (template_coverage.png).

template_degree_visited.txt
This data dependency means how many avarage visits have all vertices with the given degree. Meaning of the degree for directed graphs is sum of both in and out degrees. There are two more files for directed graphs: template_in_degree_visited.txt and template_out_degree_visited.txt.

template_degree_time.txt
This data dependency means in what avarage time all vertices with the given degree were first visited by the random walk. Meaning of the degree for directed graphs is sum of both in and out degrees. There are two more files for directed graphs: template_in_degree_time.txt and template_out_degree_time.txt.

template_degree_time_length.txt
This is data dependency of a ratio of the first visit time and length from the starting point of the random walk on the degree. The length of a vertex from the starting point is computed by BFS algorithm. Meaning of the degree for directed graphs is sum of both in and out degrees. There are two more files for directed graphs: template_in_degree_time_length.txt and template_out_degree_time_length.txt.
