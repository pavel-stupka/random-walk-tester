RWTester
software for testing a random walk

Graph generating

This tutorial will teach you how to use RWTester to generate graphs. RWTester can generate several types of graphs. These are:

  • Complete graphs
  • Scale-free graphs
  • Trees
  • Random graphs (since version 0.1)

More types of graphs are planned to be implemented.

In general

If you want to use RWTester to generate a graph you have to use a -generate option. An argument of this option is a regular expression that describes required graph. Output of this action is a graph saved to the file with a name of the given regular expression and with the suffix .graph. Let's have a look at the following example: 

rwtester -generate K50

This command will generate a complete graph with 50 vertices and saves it to the file named K50.graph.If you do not want to use a default name for the output, you can use -template option as follows:

rwtester -generate K50 -template complete

This command will generate a complete graph with 50 vertices and saves it to the file named complete.graph.

Complete graphs

Read more about complete graphs here. The regular expression for complete graphs starts with the 'K' character followed by an integer saying how many vertices the graph should be built from. The regular expression looks like this: 

Ky
  • y value — how many vertices should be generated

Scale-free graphs

Every new vertex is connected to exactly x allready existing vertices that are chosen randomly. This is the way to build a scale-free network. The regular expression for these graphs starts with 'SF' characters followed by two integers. The regular expression looks like this: 

SFx-y
  • x value — to how many vertices the new vertex should be randomly connected
  • y value — how many vertices should be generated

Trees

Read more about trees here. The regular expression for trees starts with the 'T' character followed by two integers. The regular expression looks like this: 

Tx-y
  • x value — arity of a tree (number of childrens of every node)
  • y value — depth of a tree

For example T2-3 generates a binary tree with a depth equal to three. That means the graph has 15 vertices and 14 edges.

Random graphs graphs

First the program generates all vertices and then all edges that every edge connects two randomly selected vertices. See random graphs. The regular expression for these graphs starts with 'R' character followed by two integers. The regular expression looks like this: 

Rx-y
  • x value — total number of vertices
  • y value — total number of edges that are choosen randomly
